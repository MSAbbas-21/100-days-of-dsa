# 📘 Linear Search - Theory

## Definition
Linear Search is a searching algorithm that checks each element one by one until the target element is found or the array ends.

**Also Known As:** Sequential Search

---

## Working

1. Start from the first element.
2. Compare it with the target.
3. If matched, return the result.
4. Otherwise, move to the next element.
5. Repeat until the target is found or the array ends.

---

## Characteristics

- Checks elements sequentially.
- Works on both sorted and unsorted arrays.
- Simple and easy to implement.
- No extra memory required.

---

## Time Complexity

- **Best Case:** `O(1)`
- **Average Case:** `O(n)`
- **Worst Case:** `O(n)`

---

## Space Complexity

- **O(1)**

---

## Advantages

- Easy to understand.
- Works with unsorted data.
- No extra space needed.

---

## Disadvantages

- Slow for large datasets.
- Checks elements one by one.

---

## Key Points

- Also called **Sequential Search**.
- Traverses the array from left to right.
- Stops when the target is found.
- Suitable for small datasets.
